ARCHITECTURE OVERVIEW
Before writing any code, understand the full data and request flow:
STEP 1 (Voice Intake)
User mic → MediaRecorder (audio/webm) → POST /api/transcribe (FormData) 
→ OpenAI Whisper → transcript text → POST /api/chat (Claude) 
→ coaching response text → POST /api/speak (ElevenLabs) 
→ mp3 audio buffer → Blob URL → <audio> element plays

End of Step 1:
Full transcript → POST /api/extract-profile (Claude)
→ structured profile JSON → localStorage → navigate to /dashboard

STEP 2 (Dashboard)
localStorage profile JSON → React components render pyramid, life map, quadrant
Profile JSON → POST /api/narrative (Claude) → narrative text 
→ POST /api/speak (ElevenLabs) → mp3 → Blob URL → <audio> element

STEP 3 (Coach Chat)
Profile JSON from localStorage → Claude system prompt context
Same voice pipeline as Step 1

STEP 4 (Strategic Plan)
Full conversation + profile → POST /api/generate-plan (Claude)
→ plan text → POST /api/speak (ElevenLabs) → mp3 → <audio> element

BACKEND — EXPRESS SERVER
The Express server must expose exactly these endpoints. Add CORS middleware at the top:
javascriptconst cors = require('cors');
app.use(cors());
app.use(express.json({ limit: '10mb' }));
Endpoint 1: POST /api/transcribe

Accepts: multipart/form-data with field audio (audio/webm file)
Use multer with storage: multer.memoryStorage() to hold file in memory
Send req.file.buffer to OpenAI Whisper:

javascriptconst { OpenAI } = require('openai');
const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });
const File = require('node:buffer').File; // Node 20+
const file = new File([req.file.buffer], 'audio.webm', { type: 'audio/webm' });
const result = await openai.audio.transcriptions.create({
  model: 'whisper-1',
  file: file,
});
res.json({ transcript: result.text });

Returns: { transcript: string }

Endpoint 2: POST /api/speak

Accepts: { text: string }
Calls ElevenLabs TTS REST API:

javascriptconst response = await fetch(
  `https://api.elevenlabs.io/v1/text-to-speech/${process.env.ELEVENLABS_VOICE_ID}`,
  {
    method: 'POST',
    headers: {
      'xi-api-key': process.env.ELEVENLABS_API_KEY,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      text: req.body.text,
      model_id: 'eleven_monolingual_v1',
      voice_settings: { stability: 0.5, similarity_boost: 0.75 }
    })
  }
);
const audioBuffer = await response.arrayBuffer();
res.set('Content-Type', 'audio/mpeg');
res.send(Buffer.from(audioBuffer));

Returns: raw mp3 binary (Content-Type: audio/mpeg)

Endpoint 3: POST /api/chat

Accepts: { messages: [{role, content}], systemPrompt: string }
Calls Claude claude-sonnet-4-6
Returns: { response: string }

Endpoint 4: POST /api/extract-profile

Accepts: { transcript: string }
Calls Claude with this instruction: "You are extracting structured data from a life coaching conversation transcript. Return ONLY valid JSON matching this exact schema, nothing else, no markdown, no explanation:"
Returns the following JSON schema — Claude must populate every field:

json{
  "firstName": "string",
  "dimensions": [
    {
      "name": "string",
      "importance": "high|medium|low",
      "thriving": "high|medium|low",
      "tier": "foundational|growth|meaning",
      "themes": ["string"],
      "notableQuote": "string"
    }
  ],
  "lifeEvents": [
    {
      "label": "string",
      "approximateYear": "number or null",
      "chapterName": "string",
      "actualizationLevel": "number between 20 and 95",
      "type": "peak|low|transition|milestone"
    }
  ],
  "values": ["string"],
  "purposeThemes": ["string"],
  "overallNarrative": "string of 5-8 sentences telling the user's story"
}
Important rules for Claude when populating this schema:

actualizationLevel for life events: use 20–95 range only. Overall trend must be upward across the timeline despite dips. Low points should be 25–45, transitions 40–60, milestones 60–80, peaks 70–95. Never assign 0 or 100. The principle: challenges are part of the upward arc.
tier for dimensions: Physical and Financial default to "foundational" unless conversation strongly suggests otherwise. Mental & Emotional and Social & Relationships default to "growth". Career defaults to "meaning". Any "other" dimension — AI assigns tier based on what the user described.
Dimensions array must contain all 5 standard dimensions plus any "other" dimensions mentioned
Returns: { profile: <JSON object> } — parse and store entire profile object in localStorage under key tayo_profile

Endpoint 5: POST /api/narrative

Accepts: { profile: object }
Calls Claude: "Write a warm, honest, empowering narrative of 5–8 sentences in second person ('You are...') based on this profile. Reference specific dimensions, life events, and themes. Apply the principle that over the long arc, this person is moving toward their full potential. Be specific, not generic."
Returns: { narrative: string }

Endpoint 6: POST /api/generate-plan

Accepts: { profile: object, conversationHistory: array }
Calls Claude with existing strategic plan system prompt (from v1)
Returns: { plan: string }

**FRONTEND VOICE STATE MACHINE**

The voice interface has exactly these states. Never allow transitions outside this flow:
```
IDLE
  → [page loads, AI greeting ready] → AI_SPEAKING

AI_SPEAKING
  → [audio ends] → USER_PROMPT (show mic button)

USER_PROMPT
  → [user taps mic] → USER_RECORDING

USER_RECORDING
  → [user taps mic again] → PROCESSING

PROCESSING
  → [transcript + Claude response + ElevenLabs audio ready] → AI_SPEAKING
  → [error] → ERROR (show retry button, return to USER_PROMPT)

ERROR
  → [retry] → USER_PROMPT
Audio recording implementation:
javascript// Start recording
const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
const mediaRecorder = new MediaRecorder(stream, { mimeType: 'audio/webm' });
const chunks = [];
mediaRecorder.ondataavailable = e => chunks.push(e.data);
mediaRecorder.onstop = async () => {
  const blob = new Blob(chunks, { type: 'audio/webm' });
  const formData = new FormData();
  formData.append('audio', blob, 'recording.webm');
  const res = await fetch('/api/transcribe', { method: 'POST', body: formData });
  const { transcript } = await res.json();
  // then send transcript to /api/chat
};
mediaRecorder.start();
Audio playback from ElevenLabs:
javascriptconst res = await fetch('/api/speak', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ text: aiResponse })
});
const blob = await res.blob();
const url = URL.createObjectURL(blob);
const audio = new Audio(url);
audio.onended = () => {
  URL.revokeObjectURL(url); // clean up memory
  setVoiceState('USER_PROMPT');
};
audio.play();
setVoiceState('AI_SPEAKING');
```

**Conversation phases — Claude system prompt:**
```
You are Tayo, a warm and skilled life coach conducting a voice-based intake. 
Your goal is to gather everything needed to build a whole-person profile. 
Move through these 6 phases naturally — do not announce phase changes:

Phase 1 — Welcome: Introduce yourself warmly. Ask their name. Explain you'll 
explore their life together. Make them feel safe.

Phase 2 — Life journey: Ask about their story. What chapters would they name 
if their life were a book? What has been most challenging, fulfilling, rewarding? 
How have they grown in recent years? What would they call their highest point so 
far, and their lowest? Gather approximate timeframes for key events.

Phase 3 — Five dimensions (one at a time): Mental & Emotional, Career, Physical, 
Social & Relationships, Financial. For each: how are they doing, how important is 
it, why does it matter, what is driving their current state, what has shifted recently. 
Use an asset and strength-based lens — surface resilience and growth, not just 
struggles.

Phase 4 — Other dimensions: Ask if there are other important areas of their life 
not yet covered. If yes, explore with the same depth.

Phase 5 — Values and purpose: What matters most at the deepest level? What do 
they want their life to stand for? What does their best self look like?

Phase 6 — Close: Reflect back what you heard in 3–4 warm sentences. Tell them 
you are now building their dashboard and invite them to continue.

Rules: One question at a time. Never read questions verbatim — have a real 
conversation. Build trust before going deep. Always frame through the lens that 
over the long arc of a life, people move toward self-actualization. Challenges are 
not evidence of going backward — they are often the path forward. Keep responses 
under 80 words so audio stays conversational.
End of conversation — profile extraction:
When Phase 6 closes, show a loading screen ("Building your dashboard...") and:

await fetch('/api/extract-profile', { body: JSON.stringify({ transcript: fullTranscript }) })

TAB 3 — "Where to Journey Next"
2×2 quadrant grid. Data from profile.dimensions.
Quadrant assignment logic:
javascriptconst getQuadrant = (dim) => {
  const hiImportance = dim.importance === 'high';
  const hiThriving = dim.thriving === 'high';
  if (hiImportance && hiThriving) return 'strengths';
  if (hiImportance && !hiThriving) return 'focus';
  if (!hiImportance && hiThriving) return 'surplus';
  return 'dormant';
};
```
