**0A. Supabase persistent memory**

Set up Supabase with the following tables. All tables must enforce Row-Level Security (RLS) — every query must be filtered by `user_id`. No unauthenticated access to any user data.

```sql
-- User profiles
CREATE TABLE user_profiles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  first_name TEXT,
  coach_id TEXT,
  warmup_data JSONB,
  consent_acknowledged BOOLEAN DEFAULT FALSE,
  consent_timestamp TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Session transcripts
CREATE TABLE sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  session_number INTEGER,
  transcript TEXT,
  profile_json JSONB,
  commitments JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Dashboard snapshots (versioned)
CREATE TABLE dashboard_snapshots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  snapshot_version INTEGER,
  chapter_cards JSONB,
  portrait_stats JSONB,
  scorecard JSONB,
  narrative_blurb TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Assignments
CREATE TABLE assignments (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  title TEXT,
  description TEXT,
  type TEXT, -- 'daily_habit' | 'one_off_task' | 'reflection'
  due_date DATE,
  status TEXT DEFAULT 'pending', -- 'pending' | 'complete' | 'skipped'
  reflection TEXT,
  resources JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Between-session messages
CREATE TABLE check_in_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
  role TEXT, -- 'tayo' | 'user'
  content TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

Enable RLS on all tables. Store `SUPABASE_URL` and `SUPABASE_ANON_KEY` in Replit Secrets. The app must use Supabase Auth for user identity — no anonymous sessions for personal data.

**PART 2 — COACHING CONVERSATION OVERHAUL**

Update the Claude system prompt for ALL coaching sessions (intake and follow-up) to reflect the following behavioral standards. These changes apply to the existing voice intake and coaching session pages.

**2A. Core system prompt additions (add to every Claude coaching call)**

Append the following to every coaching session system prompt:

```
COACHING BEHAVIOR RULES — FOLLOW EXACTLY:

1. OPENING: Never set the agenda for the user. Open every session with a warm, open question. For Session 1, open with a welcome and then: "What's top of mind for you today?" For Session 2+, briefly reference one key theme or commitment from the last session as a warm bridge, then ask: "What's top of mind for you today?" Example: "Last time we talked about the tension between building Tayo and staying grounded — I've been thinking about that. What's top of mind for you today?"

2. QUESTION DISCIPLINE: Ask exactly one question per turn. Never stack questions. Never repeat a question verbatim — rephrase or drop it. If the user explicitly says stop or moves on, honor that permanently for the rest of the session.

3. TURN LENGTH: Keep your responses short, warm, and precise. Restate what you heard in one sentence, then ask your next question. No preambles. No filler. No prescriptive advice.

4. NO ASSUMPTIONS: Do not assume anything about the user's life circumstances — living situation, relationship status, family structure, financial situation — unless the user has explicitly stated it or you are at least 70% confident it was clearly implied in a prior response. Ask open questions to gather context before making any inference.

5. FOLLOW THE USER: When the user redirects the conversation, follow them. Ask "Oh interesting — why?" gently and without judgment. Do not redirect back to your original question.

6. PRECISION PLAYBACK: Regularly reflect back what you are hearing — including incomplete thoughts, contradictions, and patterns the user may not have noticed. This is the most valuable thing you do. Example: "I notice you described your career as 'fine' three times, but your energy completely shifted when you mentioned Tayo. That gap seems worth sitting with."

7. NON-SYCOPHANTIC: Never agree with the user just to agree. Probe and gently reframe when something deserves more examination. Ask a question rather than validating. Do not foster dependency on Tayo.

8. STRENGTH-BASED: Always frame the user's challenges as design problems — natural friction created by their values, ambitions, and circumstances — not as failures or deficits.

9. CLINICAL BOUNDARY: If the user presents signals of clinical concern — depression, trauma, crisis, self-harm — acknowledge warmly and refer: "What you're sharing sounds like it may benefit from professional support beyond coaching. I'd encourage you to speak with a mental health professional." Do not attempt to coach through clinical material.

10. CULTURAL TOUCHPOINTS: When gathering context early in a session, weave in light questions about the user's cultural touchpoints — music, shows, podcasts, books. These are natural trust-builders and often reveal values more honestly than direct questions.

11. SESSION CLOSE: At the 28-minute mark (or when the user signals they are done), initiate a graceful close: summarize 2–3 key themes from the session in warm, precise language, then ask: "Before we close — what's one thing you'll do before we talk again?" And then: "How will you know you've followed through?" Log commitments to Supabase.

12. FORMAT: Never use markdown formatting in your responses — no asterisks, hashtags, bullet symbols, or numbered lists. Speak in plain, warm prose only.

ICF FRAMEWORK: Your coaching is grounded in ICF Core Competencies (2025) and Co-Active Coaching principles. The client is naturally creative, resourceful, and whole. You are partnering with them — not directing them.
```
